ISSUE REPORTED:
  The connection switch didn't actually disconnect the interface.
  User clicked disconnect on wlp3s0 but it remained connected.

ROOT CAUSE:
  Disconnect function only flushed IP addresses but didn't:
  - Stop DHCP client processes (dhcpcd/dhclient)
  - Release DHCP leases properly
  - Verify disconnection

═══════════════════════════════════════════════════════════════

FIXED FUNCTIONALITY:

1. STATUS SWITCH (First switch)
   Purpose: Bring interface UP or DOWN
   
   UP (ON):
   ✓ Executes: interface.up()
   ✓ Brings link layer up
   ✓ Interface becomes operational
   
   DOWN (OFF):
   ✓ Executes: interface.down()
   ✓ Brings link layer down
   ✓ Interface becomes non-operational

2. CONNECTION SWITCH (Second switch)
   Purpose: Connect/Disconnect (DHCP or static IP)
   
   CONNECT (ON):
   ✓ Detects backend (NetworkManager/systemd/manual)
   ✓ Requests DHCP lease
   ✓ Waits for IP assignment
   ✓ Shows success with IP address
   ✓ Shows which backend was used
   
   DISCONNECT (OFF):
   ✓ Detects backend
   ✓ Stops DHCP client (dhcpcd -k / dhclient -r)
   ✓ Releases DHCP lease
   ✓ Flushes all IP addresses
   ✓ Verifies disconnection
   ✓ Shows warning if IP remains

DISCONNECT PROCESS:

For NetworkManager-managed interfaces:
  1. Run: nmcli device disconnect <interface>
  2. NetworkManager handles lease release
  3. IP automatically cleared

For Manual interfaces:
  1. Detect DHCP client (dhcpcd/dhclient/udhcpc)
  2. Stop client with proper release:
     - dhcpcd: dhcpcd -k <interface>
     - dhclient: dhclient -r <interface>
  3. Flush IP addresses: ip addr flush dev <interface>
  4. Verify IP is cleared
  5. Return success/failure

Connection Switch ON:

Connection Switch OFF:

  • IP address removed

Status Switch affects link layer only:

  • DOWN = Interface disabled

SWITCH BEHAVIOR SUMMARY:

STATUS SWITCH:
  Controls: Link layer (UP/DOWN)
  Commands: interface.up() / interface.down()
  Effect: Interface operational state
  Typical use: Enable/disable hardware

CONNECTION SWITCH:
  Controls: Network connection (IP assignment)
  Commands: DHCP client / nmcli / networkctl
  Effect: IP address assignment/removal
  Typical use: Connect/disconnect to network

Both switches work independently:
  • Can be UP but disconnected (no IP)
  • Can be DOWN (connection switch disabled)
  • Can be UP and connected (normal operation)
